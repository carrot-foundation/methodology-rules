# Project Documentation Index — methodology-rules

_Primary entry point for AI-assisted development. Generated by BMAD document-project (initial deep scan), 2026-06-23._

## Project Overview

- **Type:** Nx monorepo (single cohesive **backend** part)
- **Primary Language:** TypeScript 5.9.3 on Node 24.14.1
- **Architecture:** Serverless, function-per-rule — AWS Lambda rule processors for the BOLD methodology framework
- **Verdict contract:** `RuleInput` → `RuleOutput` (`PASSED` / `FAILED` / `REVIEW_REQUIRED`), reported to Smaug via SigV4

## Quick Reference

- **Tech Stack:** TypeScript · AWS Lambda · Nx 22.6.1 · pnpm 10.18.3 · Vitest 4.1.0 · Zod 4.3.6 · bignumber.js · geolib · Textract · Sentry · pino
- **Entry Point (per rule):** `{rule}.lambda.ts` → `wrapRuleIntoLambdaHandler(new {Rule}Processor())`
- **Architecture Pattern:** Layered — `apps → rule-processors → libs/shared` (Nx tag-enforced)
- **Scale:** ~100 Nx projects · 23 rule-processor projects · 3 methodology frameworks (BOLD, BOLD Carbon, BOLD Recycling)

## Getting Started

```bash
nvm use            # Node 24.14.1
pnpm install       # pnpm only
pnpm test:affected # Vitest
```

- New rule: `pnpm create-rule <name> <scope> <description>` then `pnpm apply-methodology-rule …`. Never hand-create processor files.
- Run locally: `pnpm run-rule -- <args>`.
- See the [Development Guide](./development-guide.md) for conventions and the codegen sync steps.
