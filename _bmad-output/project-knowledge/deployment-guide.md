## Deployment Model

Each rule processor is bundled and deployed as an **independent AWS Lambda**. Bundling uses `@nx/webpack` (custom config at `.webpack/webpack.nx.config.js`); CI builds only **Nx-affected** Lambdas and uploads the artifacts to S3 per environment.

## Build & Package (local)

```bash
pnpm build-lambda <project>      # nx build-lambda â€” webpack bundle (tsc compiler)
# nx package-lambda <project>    # zips the bundle (depends on build-lambda)
```

Cacheable Nx targets: `build-lambda`, `package-lambda`, `test`, `lint`, `ts`. Remote cache: S3 bucket `carrot-nx-cache` (us-east-1).

## CI/CD Pipeline (`.github/workflows/`)

| Workflow | Role |
|---|---|

### `deploy.yaml` shape

- Triggered via `workflow_call` with secrets: `AWS_ACCOUNT_ID_PRODUCTION`, `AWS_ACCOUNT_ID_DEVELOPMENT`, `ADMIN_GITHUB_TOKEN`, `SLACK_WEBHOOK_URL`, `nx_key`.
- `AWS_REGION: us-east-1`.
- Matrix over `environment: [production, development]`.
- Uses the `prepare-to-run-nx` composite action (full git history for Nx affected), then builds and uploads each affected Lambda artifact to S3.

## Runtime IAM / Egress

At runtime, a rule Lambda assumes a cross-account role (`SMAUG_API_GATEWAY_ASSUME_ROLE_ARN`) via **STS**, then signs its result POST to Smaug with **SigV4**. This is how `reportRuleResults` authenticates to the Smaug `AUDIT_URL`.

## Versioning

```bash
pnpm bump:applications   # bump deployable application versions
pnpm bump:rules          # bump individual rule versions
```

Rule `version` lives in each `{rule}.rule-definition.ts` and flows into generated manifests.

## Error Handling at Runtime

The Lambda wrapper lets unrecoverable errors **propagate** so AWS retry / DLQ mechanisms engage. Failures set Sentry tags (`documentId`, `documentKeyPrefix`, `requestId`, `ruleName`). Do not catch-and-swallow in handlers.

## Manual Artifact Upload

`pnpm upload-lambdas-and-metadata` uploads Lambda artifacts and metadata outside the standard CI path (use with care; CI is the normal route).
