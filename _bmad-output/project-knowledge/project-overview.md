# Project Overview — methodology-rules

_Generated by BMAD document-project (initial deep scan), 2026-06-23._

## Purpose

**methodology-rules** is the rule-processor engine for Carrot Foundation's **BOLD methodology framework**. Each rule processor is an independent **AWS Lambda** that validates one aspect of a waste-management lifecycle document (MassID, MassID Certificate, Credit Order), then reports an audit result (`PASSED` / `FAILED` / `REVIEW_REQUIRED`) back to the Smaug API.

A rule receives a reference to an S3-stored document, traverses the document relationship graph to gather the documents it needs, applies its validation logic, and posts a signed result. Rules are pure validators: they never mutate documents.

## Repository Type

**Nx monorepo** — a single, cohesive TypeScript codebase. ~100 Nx projects: 23 rule-processor projects plus ~20 shared libraries, 3 methodology rule-sets, and CLI/codegen tooling. One language, one build system, one deploy pipeline; not a multi-part (client/server) repository.

## Architecture Type

**Service / function-per-rule (serverless).** Layered dependency model:

```
apps/methodologies/*  (Lambda entry points / deployable units)
        │
        ▼
libs/methodologies/{bold,bold-carbon,bold-recycling}/rule-processors/*  (validation logic)
        │
        ▼
libs/shared/*  (rule, lambda, app, document, methodologies, helpers, testing, …)
```

Processors depend on `libs/shared/*` only — never on other processors. See [architecture.md](./architecture.md).

## Methodology Frameworks

- **BOLD** (`libs/methodologies/bold`) — MassID lifecycle: 20 MassID rule processors + 1 MassID-Certificate processor + 1 Credit-Order processor.
- **BOLD Carbon** (`libs/methodologies/bold-carbon`) — Carbon Certificate rule set.
- **BOLD Recycling** (`libs/methodologies/bold-recycling`) — Recycling Credit Order rule set.
